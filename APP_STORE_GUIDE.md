# Apple App Store Publication Checklist

## Account & Setup

- [ ] Join Apple Developer Program ($99/year)
- [ ] Create Apple ID
- [ ] Enable App Specific Passwords
- [ ] Accept Developer Agreement
- [ ] Configure App Store Connect

## App Information

### Basic Info
- [ ] App name: "Block Description"
- [ ] Subtitle (optional, 30 chars)
- [ ] Category: Games > Arcade
- [ ] Subcategory: Action
- [ ] Privacy Policy URL (REQUIRED)
- [ ] Support URL

### Description
- [ ] Promotional text (170 chars) - What's new
- [ ] Description (4000 chars max)
- [ ] Keywords (100 chars)
- [ ] Support URL
- [ ] Privacy Policy URL

### Graphics & Media

#### App Icon
- [ ] Size: 1024x1024 PNG (no transparency)
- [ ] No rounded corners
- [ ] RGB color space

#### Screenshots
- [ ] Minimum 2, maximum 5 per device
- [ ] iPhone 6.7": 2532x1170 PNG
- [ ] iPhone 5.5": 1242x2208 PNG
- [ ] iPad: 2048x2732 PNG
- [ ] Add captions describing features

#### App Preview (optional)
- [ ] Video: 15-30 seconds
- [ ] Demonstrate core gameplay
- [ ] Format: MP4 or MOV
- [ ] Resolution: 1280x720 (portrait)

## Age Rating

- [ ] Complete EULA
- [ ] Select content descriptors
- [ ] Set age rating (4+, 12+, 17+)
- [ ] Verify COPPA compliance

## Build & Submission

### Build Configuration
- [ ] Build number (must increment)
- [ ] Version number (1.0.0 format)
- [ ] Minimum iOS version: 13.4+
- [ ] Supported devices

### Compliance

- [ ] No prohibited content
- [ ] Proper age rating
- [ ] Working URLs
- [ ] No private APIs
- [ ] Proper app behavior
- [ ] Data privacy compliance

### Testing

- [ ] Test on iPhone X/11/12/13/14/15
- [ ] Test on iPad
- [ ] Test in landscape and portrait
- [ ] Check for crashes
- [ ] Verify game performance
- [ ] Test offline functionality
- [ ] Check memory usage

## Store Listing

1. **Create new app** in App Store Connect
   - Bundle ID: com.blockdescription.game
   - App type: Game

2. **Fill in app information**
   - All required fields marked
   - Compelling description
   - Good screenshots

3. **Add version information**
   - Version number: 1.0.0
   - Release notes
   - Copyright

4. **Upload build**
   - Use EAS submit
   - Or use Xcode / Transporter
   - Build goes to TestFlight first

5. **Submit for review**
   - Review takes 1-2 days
   - Can be rejected and resubmitted
   - Check App Store Review Guidelines

## Pricing

- [ ] Set as Free app
- [ ] No In-App Purchases (unless adding later)
- [ ] Available in all regions

## Review Guidelines Compliance

- [ ] No crashes on startup
- [ ] Functional core gameplay
- [ ] UI legible and responsive
- [ ] Proper use of APIs
- [ ] No misleading marketing
- [ ] Age-appropriate content
- [ ] Working demo/tutorial

## Submission Checklist

Before submitting to review:

1. [ ] Complete app information
2. [ ] Add all screenshots (min 2)
3. [ ] Upload build via TestFlight
4. [ ] Test build on device
5. [ ] Accept Developer Agreement
6. [ ] Select export compliance
7. [ ] Enable third-party services (if used)
8. [ ] Submit for review

## After Approval

- [ ] Schedule release date
- [ ] Write release notes
- [ ] Promote on social media
- [ ] Monitor reviews and ratings
- [ ] Respond to user feedback
- [ ] Plan future updates

## Troubleshooting

### Build Rejected
- Check crash logs in TestFlight
- Verify compliance with guidelines
- Test on multiple devices
- Ensure proper permissions

### Review Rejected
- Read rejection reason carefully
- Fix issues mentioned
- Resubmit with explanation
- Contact Apple if confused

### App Store Optimization
- Update screenshots regularly
- Refresh promotional text
- Monitor app analytics
- A/B test different descriptions
